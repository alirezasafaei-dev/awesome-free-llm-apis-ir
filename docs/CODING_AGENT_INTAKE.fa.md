# راهنمای ثبت ابزارهای کدنویسی CLI و مدل‌های رایگان

این مخزن در اصل «API رایگان LLM» را فهرست می‌کند، اما ابزارهای جدید کدنویسی مثل OpenCode، MiMo Code، Hermes و ابزارهای مشابه معمولاً دو لایه دارند:

1. **Provider/API یا Gateway**: سرویس کلید API، Base URL، مدل، سهمیه و محدودیت مصرف دارد.
2. **CLI/Agent/Editor**: ابزار مصرف‌کننده است و ممکن است از چند Provider استفاده کند، بدون اینکه خودش API عمومی رایگان باشد.

برای اینکه داده‌ها دقیق و قابل راستی‌آزمایی بمانند، این دو لایه را جدا ثبت می‌کنیم.

## معیار ورود به `data/providers`

یک ابزار یا سرویس فقط وقتی فایل Provider می‌گیرد که این موارد قابل اثبات باشند:

- Base URL یا روش رسمی فراخوانی API برای مصرف برنامه‌نویسی وجود داشته باشد.
- رایگان بودن مدل/سهمیه با مدرک رسمی یا تست تاریخ‌دار قابل بررسی باشد.
- محدودیت‌ها تا حد ممکن روشن باشند؛ اگر نامحدود ادعا می‌شود، باید قید «سیاست مصرف منصفانه»، «محدودیت زمانی» یا «نامشخص» ثبت شود.
- نیاز یا عدم نیاز به کارت بانکی، حساب، شماره تلفن، OAuth یا Billing مشخص شود.
- وضعیت ایران جداگانه و با همان روش پروژه ثبت شود؛ CLI بودن به معنی دسترسی از ایران نیست.

اگر فقط CLI وجود دارد و API عمومی رایگان مستند نیست، آن مورد نباید Provider مستقل شود؛ در عوض در راهنمای ابزارها/نمونه‌استفاده معرفی می‌شود.

## پیشنهاد ثبت موارد مطرح‌شده

| مورد | وضعیت پیشنهادی | دلیل | کار بعدی |
|---|---|---|---|
| OpenCode | «ابزار/CLI» + در صورت اثبات OpenCode Zen به‌عنوان Provider جدا | خود OpenCode یک Agent متن‌باز و مصرف‌کنندهٔ مدل‌هاست؛ مستنداتش می‌گوید از Providerهای متعدد و Models.dev استفاده می‌کند. Zen می‌تواند Provider جدا باشد، اما باید API، رایگان بودن مدل‌ها و محدودیت‌ها جداگانه ثبت شود. | مدرک رسمی مدل‌های رایگان Zen، Base URL، نیاز Billing و Limitها را جمع‌آوری کنیم. |
| MiMo Code | «ابزار/CLI»؛ MiMo API یا MiMo Auto فقط در صورت اثبات سهمیه | MiMo Code ابزار کدنویسی ترمینالی است. در مستندات/بلاگ، MiMo Auto رایگانِ محدودزمانی و MiMo API/Token Plan ذکر شده؛ بنابراین «بینهایت» نباید بدون مدرک رسمی وارد شود. | برای MiMo Auto تاریخ پایان/سیاست مصرف منصفانه و برای MiMo API Base URL و پلن رایگان را پیدا کنیم. |
| Hermes Agent | «ابزار/Agent» نه Provider | Hermes خودش چارچوب/Agent است و Providerها را از طریق Wizard یا تنظیمات وصل می‌کند؛ مدل رایگان معمولاً از OpenRouter/NVIDIA/OpenCode Zen و دیگر Gatewayها می‌آید. | راهنمای استفاده از Providerهای موجود پروژه در Hermes اضافه شود، نه Provider جدید. |
| Devine/Devin | «نیازمند شفاف‌سازی» | نام دقیق، لینک رسمی و نسبت آن با API رایگان روشن نیست؛ نتایج عمومی بیشتر به Devin/Devineهای نامرتبط اشاره می‌کنند. | قبل از ورود، لینک رسمی، مدل‌ها، API و محدودیت رایگان لازم است. |

## قالب پیشنهادی معرفی در README یا وب‌سایت

برای این دسته بهتر است یک بخش جدا با عنوان «ابزارهای CLI سازگار با Providerهای رایگان» داشته باشیم، نه اینکه همه را کنار APIها بیاوریم:

```md
### ابزارهای CLI برای استفاده از Providerهای رایگان

این ابزارها خودشان الزاماً Provider رایگان نیستند، اما می‌توانند با APIهای رایگان/دارای مدل رایگان این فهرست کار کنند.

| ابزار | نوع | Providerهای قابل اتصال | رایگان داخلی | نکته ایران |
|---|---|---|---|---|
| OpenCode | CLI/Agent | OpenRouter، Groq، Gemini، Ollama، OpenCode Zen و ... | نیازمند بررسی برای Zen | وضعیت به Provider انتخابی بستگی دارد |
| MiMo Code | CLI/Agent | MiMo Auto، MiMo API، تنظیمات سفارشی | محدودزمانی/نیازمند بررسی | وضعیت به مسیر ورود و Provider بستگی دارد |
| Hermes Agent | Agent/CLI | OpenRouter، NVIDIA NIM و Providerهای قابل تنظیم | ندارد؛ وابسته به Provider | وضعیت به Provider بستگی دارد |
```

## نمونهٔ تصمیم‌گیری برای افزودن Provider جدید

- اگر «OpenCode Zen» API Key، endpoint رسمی و مدل‌های `*-free` با محدودیت مشخص داشته باشد: فایل `data/providers/opencode-zen.json` اضافه شود.
- اگر «MiMo API» پلن رایگان عمومی و endpoint مستند داشته باشد: فایل `data/providers/xiaomi-mimo.json` اضافه شود.
- اگر فقط «MiMo Code با MiMo Auto رایگان محدودزمانی» وجود داشته باشد: در بخش CLI معرفی شود و Provider مستقل نشود، مگر اینکه API قابل مصرف خارج از CLI هم مستند باشد.
- اگر Hermes فقط OpenRouter یا NVIDIA را وصل می‌کند: همان Providerهای موجود را لینک کنیم و برای Hermes فایل Provider نسازیم.

## چک‌لیست اطلاعات لازم از مشارکت‌کننده

برای هر مورد جدید، Issue یا PR باید این اطلاعات را داشته باشد:

- لینک رسمی وب‌سایت و مستندات API/CLI
- نام دقیق سرویس و مدل‌های رایگان
- نوع رایگان بودن: دائمی، مدل رایگان، Trial، اعتبار ماهانه، محدودزمانی یا نامشخص
- محدودیت‌ها: RPM/RPD/TPM/TPD/Context، یا متن رسمی Fair Use
- نیاز به کارت بانکی، Billing، شماره تلفن یا OAuth
- آیا OpenAI-compatible است؟ اگر بله Base URL و مثال cURL
- وضعیت ایران: مستقیم، VPN، ثبت‌نام، یا نامشخص با تاریخ تست
- آیا داده/کد کاربر برای آموزش استفاده می‌شود یا سیاست حریم خصوصی جدا دارد؟
