# نقشه راه موتور رشد محتوای فارسی

## هدف

جذب توسعه‌دهنده فارسی‌زبان از جست‌وجوی فارسی و جامعه‌های فنی ایرانی، حل مسئله واقعی او در سایت و تبدیل بخشی از کاربران به مشارکت‌کننده، Reporter یا Star در GitHub.

## اصل انتشار

نسخه کامل و مرجع هر محتوا باید روی دامنه اصلی منتشر شود. نسخه‌های ویرگول، Telegram و LinkedIn بازنویسی مستقل‌اند و به Canonical اصلی ارجاع می‌دهند.

مسیر مطلوب:

```text
جست‌وجوی فارسی / جامعه فنی
        ↓
مقاله مسئله‌محور فارسی
        ↓
صفحه Provider یا Catalog
        ↓
اجرای نمونه‌کد
        ↓
Star / گزارش دسترسی / اصلاح داده در GitHub
```

## ستون‌های محتوایی

### ۱. انتخاب و مقایسه

- انتخاب API رایگان برای ایران
- API بدون کارت بانکی
- APIهای OpenAI-compatible
- مقایسه Providerها برای کاربرد مشخص
- تفاوت Free Tier، Trial و Credit

### ۲. آموزش عملی

- Python
- Node.js
- ساخت چت‌بات
- RAG و Embedding
- Agent و Tool Calling
- مهاجرت از OpenAI به Base URL سفارشی

### ۳. حل خطا

- خطای 429
- خطای 401 و 403
- Model not found
- Timeout و 5xx
- ثبت‌نام و محدودیت منطقه‌ای
- مدیریت Context و مصرف توکن

### ۴. داده ویژه ایران

- گزارش ماهانه وضعیت دسترسی
- Providerهای نیازمند تست
- تغییر سیاست منطقه‌ای
- تجربه‌های تاریخ‌دار کاربران
- سرویس‌های ایرانی دارای API عمومی و سهمیه اثبات‌شده

## جلوگیری از Keyword Cannibalization

هر URL باید یک Intent اصلی داشته باشد:

- `best-free-llm-api-iran`: مقایسه و انتخاب کلی
- `practical-free-llm-api-iran`: فرآیند عملی از ثبت‌نام تا تست
- `build-persian-chatbot-python-free-llm-api`: آموزش ساخت پروژه
- `llm-api-rate-limit-429`: حل خطا و Reliability

مقاله جدید نباید فقط با تغییر عنوان، محتوای صفحه قبلی را تکرار کند.
