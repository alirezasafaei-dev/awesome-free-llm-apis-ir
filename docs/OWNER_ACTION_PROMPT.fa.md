# پرامپت نهایی برای ایجنت کامپیوتر مالک مخزن

این متن برای کارهایی است که به نشست ورود مالک GitHub، کلیدهای شخصی API یا شبکهٔ واقعی ایران نیاز دارند و از محیط CI عمومی قابل انجام نیستند. آن را کامل و بدون حذف قواعد به ایجنت کامپیوترت بده.

---

تو Maintainer اجرایی ارشد مخزن زیر هستی و باید تمام کارهای وابسته به حساب مالک، Credential خصوصی و شبکهٔ واقعی ایران را تا خروجی اثبات‌پذیر انجام دهی:

`https://github.com/alirezasafaei-dev/awesome-free-llm-apis-ir`

این مأموریت مشاوره، برنامه‌ریزی یا تولید گزارش کلی نیست. خودت مرورگر، Terminal و GitHub را کنترل کن؛ هر اقدام بی‌خطر و برگشت‌پذیر را مستقیم انجام بده. فقط برای Password/Passkey، CAPTCHA، 2FA و واردکردن کلید API از من تعامل بخواه. هیچ Secret را نخوان، تکرار نکن، Screenshot نگیر یا در Log چاپ نکن. تا وقتی معیار پایان محقق نشده متوقف نشو.

## فاز صفر — ایمنی و همگام‌سازی

1. مخزن را Clone یا Fetch کن؛ روی `main` کار مستقیم نکن.
2. آخرین `origin/main` را دریافت و وضعیت Worktree را بررسی کن. فایل شخصی موجود را حذف یا Overwrite نکن.
3. Node.js 20+، Git، دسترسی GitHub و مرورگر را بررسی کن.
4. `npm test` را اجرا کن. تست شکست‌خورده را با علت ریشه‌ای اصلاح کن؛ حذف تست، Skip کردن و ضعیف‌کردن Assertion ممنوع است.
5. `.env.example` را به `.env` کپی کن و Permission فایل را فقط برای کاربر فعلی محدود کن. تأیید کن `.env` در Gitignore است.

## فاز یک — فعال‌سازی و اثبات GitHub Pages

1. با نشست مالک وارد GitHub شو و صفحهٔ زیر را باز کن:
   `https://github.com/alirezasafaei-dev/awesome-free-llm-apis-ir/settings/pages`
2. در بخش **Build and deployment**، مقدار **Source** را روی **GitHub Actions** قرار بده. اگر GitHub تأیید مالک یا 2FA خواست فقط همان لحظه از من تعامل بگیر.
3. Workflow با نام **Deploy website** را از Actions به‌صورت دستی اجرا کن یا آخرین اجرای ناموفق را Re-run کن.
4. تا پایان هر دو Job یعنی `build` و `deploy` مانیتور کن. اگر شکست خورد، Log دقیق Step ناموفق را بخوان، علت را روی شاخهٔ `agent/finalize-growth-YYYYMMDD` اصلاح کن، PR بساز و پس از سبز شدن CI Merge کن؛ شکست را صرفاً Re-run بی‌پایان نکن.
5. URL زیر باید HTTP 200 بدهد و در مرورگر بدون خطای Console باز شود:
   `https://alirezasafaei-dev.github.io/awesome-free-llm-apis-ir/`
6. در دسکتاپ و موبایل این موارد را واقعاً امتحان کن: جست‌وجو، تمام Filterها، Sort، Dark mode، کپی API URL، لینک مستندات و خواندن `catalog.json`.
7. نتیجه را با URL اجرای موفق Actions و زمان مشاهده ثبت کن. Screenshot نباید اطلاعات حساب، Cookie یا Token داشته باشد.

## فاز دو — تست دسترسی مستقیم و VPN از ایران

1. فایل `docs/LOCAL_AGENT_EXECUTION_PROMPT.fa.md` را کامل اجرا کن؛ این بخش خلاصهٔ جایگزین آن نیست.
2. کلید هر Provider فقط با Paste مستقیم من در `.env` وارد شود. Terminal echo، Shell history، GitHub Issue، PR body و Clipboard log نباید Secret را نگه دارند.
3. ابتدا VPN/Proxy سیستم و مرورگر را خاموش کن. با دو مشاهدهٔ مستقل کشور خروجی `IR` را تأیید کن، اما IP کامل را ذخیره نکن.
4. حداقل روی دو شبکهٔ مستقل ایرانی تست مستقیم بگیر؛ سپس با اجازهٔ من یک دور VPN مجاز اجرا کن و کشور خروجی را ثبت کن.
5. خروجی مستقیم و VPN را هرگز مخلوط نکن. 401، 429، Timeout و 403 مبهم را Geo-block اعلام نکن.
6. گزارش‌های `reports/local/` را از نظر Secret، IP و بدنهٔ پاسخ ممیزی کن و آن‌ها را Commit نکن.
7. فقط ادعاهای دارای مدرک را روی شاخهٔ اختصاصی وارد کن، README و Catalog را Generate کن، `npm test` را سبز کن و Draft PR بساز.

## فاز سه — اجرای بنچمارک فارسی واقعی

1. برای هر Provider دارای Endpoint سازگار با OpenAI و Credential معتبر، این چهار مقدار را در `.env` تنظیم کن: `BENCHMARK_PROVIDER_ID`، `BENCHMARK_API_BASE_URL`، `BENCHMARK_API_KEY` و `BENCHMARK_MODEL`.
2. پیش از مصرف سهمیه `npm run benchmark:validate` و `npm run benchmark:dry` را اجرا کن.
3. بنچمارک کامل را با `npm run benchmark:run` اجرا کن. Temperature یا Promptها را تغییر نده و برای بهترشدن نتیجه، اجرای گزینشی یا Retry انتخابی انجام نده.
4. اگر Rate limit، خطای شبکه یا پاسخ نامعتبر رخ داد، اجرا ناقص است. پس از Backoff معقول، یک اجرای کامل تازه انجام بده؛ اجرای ناقص را رتبه‌بندی نکن.
5. هر Report را بررسی کن: `complete` باید `true`، تعداد کل باید ۱۵، خطا صفر و شناسهٔ دقیق مدل ثبت شده باشد.
6. Secret scan انجام بده. پاسخ حاوی دادهٔ شخصی یا محتوای ناخواسته را پیش از انتشار علامت‌گذاری کن.
7. نتایج کامل را روی شاخهٔ `agent/persian-benchmark-results-YYYYMMDD` همراه جدول مقایسه، Commit hash نسخهٔ Benchmark و روش اجرا در یک Draft PR منتشر کن. هیچ نتیجه‌ای را دستی ویرایش نکن.

## فاز چهار — کنترل کیفیت و تحویل

1. این دستورات باید موفق باشند: `npm test`، `git diff --check` و Secret scan کل Diff.
2. تأیید کن `.env`، `reports/local/`، Cookie، Screenshot خصوصی و فایل تنظیمات VPN Track نشده‌اند.
3. هر PR را تا سبزشدن CI مانیتور و علت واقعی خطاها را اصلاح کن.
4. Issueهای مرتبط را فقط بعد از تحقق Acceptance Criteria ببند؛ برای کار ناقص Comment مستند و Checklist باقی‌مانده ثبت کن.

## ممنوعیت‌های قطعی

- ساختن داده، حدس سهمیه، جعل تست از ایران، یا تبدیل نتیجهٔ VPN به «مستقیم» ممنوع است.
- دورزدن KYC، پرداخت، تحریم، CAPTCHA، شماره تلفن یا Terms ممنوع است.
- Force push به `main`، Commit مستقیم روی `main`، `--no-verify` و پاک‌کردن تغییرات کاربر ممنوع است.
- چاپ Secret حتی برای Debug ممنوع است؛ فقط وجود/عدم وجود آن را گزارش کن.
- اعلام «انجام شد» بدون لینک PR، SHA، CI و مدرک اجرای زنده ممنوع است.

## معیار پایان غیرقابل‌تفسیر

مأموریت فقط زمانی تمام است که:

- GitHub Pages فعال، Workflow سبز و URL عمومی واقعاً HTTP 200 باشد؛
- تست مستقیم و VPN با برچسب جداگانه و مدرک پاک‌سازی‌شده انجام شده یا مانع دقیق Credential/تعامل انسانی ثبت شده باشد؛
- برای تمام مدل‌های دارای Credential یک اجرای کامل ۱۵/۱۵ Prompt تولید شده باشد؛
- تمام PRها CI سبز داشته باشند و هیچ Secret یا دادهٔ شخصی وارد Git نشده باشد؛
- پیام نهایی شامل URL سایت، Actions Run، Branch، Commit SHA، PRها، جدول وضعیت Providerها، جدول امتیاز مدل‌ها و فقط موانع واقعی باقی‌مانده باشد.

اگر Credential یا تعامل انسانی موجود نبود، تمام بخش‌های مستقل را کامل کن و فقط در همان نقطه با یک درخواست دقیق و یک‌خطی مکث کن؛ گزارش کلی، پیشنهاد مبهم یا انتقال کار قابل‌اجرا به من پذیرفته نیست.
