# نوع سرویس‌ها

این پروژه «API رایگان» را با ابزارهای مبتنی بر Cookie، Session یا مرورگر یکی نمی‌داند. فیلد `service_type` مرز این دسته‌ها را صریح می‌کند.

| مقدار | تعریف | نمایش در فهرست اصلی |
|---|---|:---:|
| `official_provider` | سرویس رسمی سازنده یا مالک مدل با API عمومی | ✅ |
| `official_gateway` | Gateway یا پلتفرم رسمی با قرارداد و مستندات عمومی | ✅ |
| `community_gateway` | Gateway میزبانی‌شده توسط جامعه با API و محدودیت منتشرشده | ✅ با برچسب هشدار |
| `session_bridge` | تبدیل Session، Cookie، OAuth ابزار یا HAR به API | ❌ |
| `self_hosted` | نرم‌افزار قابل میزبانی شخصی، بدون سهمیهٔ میزبانی‌شدهٔ مستقل | ❌ |

## قواعد پذیرش

- وجود UI یا Playground به‌تنهایی برای ثبت Provider کافی نیست.
- ابزارهای Session bridge باید در بخش ابزارها معرفی شوند و نباید به‌عنوان Free Tier رسمی نمایش داده شوند.
- Gateway اجتماعی باید Base URL، روش دریافت Credential، محدودیت مصرف، وضعیت نگهداری و ریسک Terms را شفاف کند.
- وضعیت ایران برای هر دسته همچنان فقط با سند رسمی یا تست تاریخ‌دار ثبت می‌شود.
- تغییر مدل یا سهمیهٔ پویا با `models.dynamic=true` و توضیح فارسی ثبت می‌شود؛ عدد حدسی وارد نمی‌شود.

## تأثیر در امتیازدهی

در رتبه‌بندی مشاور (Advisor) سایت، `community_gateway` با **۲۰- امتیاز** و `session_bridge` با **۳۰- امتیاز** جریمه می‌شوند تا در انتهای فهرست رتبه‌بندی قرار گیرند. این جریمه در همهٔ اولویت‌ها و کاربردها اعمال می‌شود.
