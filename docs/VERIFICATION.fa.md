# سطوح اعتبارسنجی

| سطح | کاربرد |
|---|---|
| `docs_verified` | سهمیه یا سیاست از مستند رسمی بررسی شده است |
| `live_verified` | درخواست واقعی با شواهد پاک‌سازی‌شده ثبت شده است |
| `community_report` | گزارش کاربر دارای تاریخ و جزئیات شبکه است، اما نگهدارنده مستقلاً تکرار نکرده است |
| `unverified` | دادهٔ اولیه است و هنوز مدرک کافی ندارد |

## قواعد وضعیت ایران

- `verified_working`، `verified_blocked` و `intermittent` بدون تست ایران پذیرفته نمی‌شوند.
- `verified_working_vpn` فقط موفقیت مسیر VPN را نشان می‌دهد و نباید «دسترسی مستقیم» ترجمه شود.
- `direct_blocked_vpn_working` به مدرک مستقل برای شکست مستقیم و موفقیت VPN نیاز دارد.
- `officially_unsupported` باید لینک سیاست یا فهرست رسمی داشته باشد.
- نبودن نام ایران در یک صفحه فقط وقتی معتبر است که همان صفحه صریحاً فهرست مناطق قابل استفاده را تعریف کند.
- خطای موقت، 429 یا قطعی یک ISP به‌تنهایی «مسدود» بودن را ثابت نمی‌کند.
- گزارش‌ها باید اسرار و اطلاعات شخصی را حذف کنند.
