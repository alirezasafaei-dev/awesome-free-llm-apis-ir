# روش‌شناسی گردآوری و راستی‌آزمایی

## اصل پایه

سه ادعای مستقل بررسی می‌شوند:

1. آیا سرویس API عمومی دارد؟
2. آیا سهمیه‌ای واقعاً بدون پرداخت ارائه می‌کند؟
3. آیا ثبت‌نام و درخواست API با IP ایران کار می‌کند؟

موفق بودن یکی، دو مورد دیگر را ثابت نمی‌کند.

## منابع سهمیه

اولویت شواهد به ترتیب زیر است:

1. مستندات رسمی قیمت و Rate Limit
2. داشبورد رسمی حساب، بدون انتشار اطلاعات خصوصی
3. پاسخ HTTP و Headerهای نرخ در تست واقعی
4. گزارش جامعه با مراحل بازتولیدپذیر

مطالب وبلاگی و شبکه‌های اجتماعی به‌تنهایی برای ثبت عدد سهمیه کافی نیستند.

## تست دسترسی ایران

تست معتبر باید حداقل شامل تاریخ، نتیجهٔ ثبت‌نام، نتیجهٔ یک درخواست کم‌هزینه، ISP یا ASN و کد وضعیت/نوع خطا باشد. شهر اختیاری است. کلید API، IP کامل، ایمیل، شماره تلفن و شناسهٔ حساب نباید منتشر شوند.

نتیجهٔ یک ISP به کل ایران تعمیم داده نمی‌شود. اگر نتایج شبکه‌ها متفاوت باشند، وضعیت `intermittent` ثبت می‌شود.

## تازگی داده

هر فایل `last_checked` و `stale_after_days` دارد. منقضی شدن به معنی غلط بودن داده نیست؛ یعنی پیش از اتکا باید دوباره بررسی شود. پیش‌فرض سهمیه‌ها ۳۰ روز است.

## محدودیت حقوقی

این پروژه روش دور زدن محدودیت جغرافیایی، تحریم، احراز هویت یا شرایط استفاده را آموزش نمی‌دهد. هدف فقط گزارش شفاف وضعیت قابل مشاهده است.
