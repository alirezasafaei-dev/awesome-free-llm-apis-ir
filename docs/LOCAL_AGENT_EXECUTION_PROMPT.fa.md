# پرامپت اجرایی ایجنت کامپیوتر محلی

متن زیر را بدون حذف بخش‌ها به ایجنت دارای دسترسی به کامپیوتر، مرورگر و اتصال واقعی ایران بدهید.

برای مأموریت یکپارچهٔ فعال‌سازی سایت، تست ایران و اجرای بنچمارک، از [پرامپت نهایی ایجنت مالک](OWNER_ACTION_PROMPT.fa.md) استفاده کنید.

---

تو مسئول اجرای کامل و اثبات‌پذیر فاز «Iran Live Verification» برای ریپوی زیر هستی:

`https://github.com/alirezasafaei-dev/awesome-free-llm-apis-ir`

نقش تو تحلیل‌گر یا مشاور نیست؛ یک ایجنت اجرایی ارشد DevOps، QA، OSINT فنی و Maintainer هستی. تا رسیدن به خروجی قابل Merge متوقف نشو. دربارهٔ کارهایی که می‌توانی مستقیماً انجام دهی سؤال نپرس. فقط در سه حالت از کاربر ورودی بخواه: ورود تعاملی به حساب، CAPTCHA/2FA، یا ارائهٔ کلید API. هیچ محدودیت جغرافیایی، احراز هویت یا شرایط استفاده‌ای را دور نزن.

## مأموریت قطعی

1. ریپو را Clone و آخرین `main` را دریافت کن.
2. محیط، Git، Node.js 20 یا جدیدتر و دسترسی GitHub را بررسی و مشکلات قابل‌حل را خودکار رفع کن.
3. شاخه‌ای با نام `agent/iran-live-verification-YYYYMMDD` بساز.
4. `npm test` را قبل از تغییر اجرا کن. شکست موجود را ثبت و اصلاح کن؛ تست را حذف یا ضعیف نکن.
5. مستندات رسمی هر یازده Provider را مجدداً بررسی کن: OpenRouter، Groq، GitHub Models، Gemini، Hugging Face، Cerebras، Cloudflare Workers AI، Mistral، SambaNova، Cohere و NVIDIA NIM.
6. برای هر سرویس مشخص کن: امکان ثبت‌نام از ایران، نیاز به کارت/پرداخت، امکان ساخت کلید، مدل رایگان فعلی، RPM/RPD/TPM/TPD و سیاست رسمی منطقه‌ای.
7. هیچ عددی را از حافظه، وبلاگ یا نتیجهٔ موتور جست‌وجو وارد نکن. هر عدد باید لینک مستقیم مستند رسمی و تاریخ مشاهده داشته باشد.
8. فایل `.env.example` را به `.env` کپی کن. کلیدها فقط در `.env` باشند. هرگز secret را در Terminal transcript، Log، Issue، Commit، Screenshot یا پیام نهایی نمایش نده.
9. تست را دو فاز اجرا کن. فاز A: VPN و Proxy را خاموش کن و با دو روش مستقل تأیید کن خروجی `IR` است. فاز B: در صورت اجازهٔ کاربر، VPN موجود و مجاز را فعال کن و کشور خروجی را ثبت کن. نتیجهٔ فاز B را هرگز دسترسی مستقیم ایران اعلام نکن. IP کامل را ذخیره یا منتشر نکن.
10. ابتدا `npm run verify:iran:dry` و سپس تست سرویس‌ها را یک‌به‌یک اجرا کن. تست‌ها باید حداقل مصرف را داشته باشند و Rate Limit را عمداً اشباع نکنند.
11. اگر ثبت‌نام نیازمند مرورگر، CAPTCHA یا 2FA بود، مرورگر را تا همان مرحله پیش ببر و فقط برای انجام تعامل انسانی مکث کن؛ بعد بلافاصله ادامه بده.
12. خطای 401 را Geo-block اعلام نکن. خطای 429 را Block اعلام نکن. Timeout یا DNS failure را بدون تکرار روی دو شبکه Block اعلام نکن. 403 فقط وقتی Geo-block است که پیام صریح منطقه‌ای داشته باشد یا کنترل همان Credential در منطقهٔ پشتیبانی‌شده موفق شده باشد.
13. برای نتیجهٔ مسدود، Credential را بدون افشای آن از یک منطقهٔ رسمی پشتیبانی‌شده کنترل کن یا مدرک رسمی صریح ارائه بده. اگر کنترل ممکن نیست، وضعیت `unknown` بماند.
14. حداقل روی دو شبکهٔ مستقل ایران تست مستقیم کن؛ ترجیحاً یک اینترنت ثابت و یک اپراتور موبایل. سپس در صورت امکان یک دور VPN جدا اجرا کن. اگر مستقیم مسدود و VPN موفق بود `direct_blocked_vpn_working` ثبت کن. اگر نتایج مستقیم شبکه‌ها متفاوت بود `intermittent` ثبت کن.
15. خروجی `reports/local/` را بازبینی کن. Secret، IP کامل و بدنهٔ پاسخ نباید وجود داشته باشد. فایل‌های Local Report را Commit نکن.
16. فقط پس از عبور از معیار مدرک، فایل Provider مربوط را به‌روزرسانی کن: `status`، `tested_from_iran`، `tested_at`، `test_method`، شبکهٔ پاک‌سازی‌شده، Evidence، `last_checked` و Notes.
17. اگر مدل یا سهمیه تغییر کرده، داده و جدول README را با `npm run generate` به‌روز کن.
18. `npm test` را تا موفقیت کامل اجرا کن. سپس `git diff --check`، جست‌وجوی secret و بررسی اینکه `.env` یا `reports/local` Track نشده‌اند انجام بده.
19. Diff را خط‌به‌خط بازبینی کن. ادعای بدون منبع، تاریخ آینده، دادهٔ شخصی، Secret، IP کامل و عدد قدیمی نباید باقی بماند.
20. فقط فایل‌های مرتبط را Commit کن. پیام Commit: `Verify LLM API access from Iranian networks`.
21. شاخه را Push و Draft PR بساز. PR باید جدول نتیجهٔ هر Provider، شبکه‌های تست‌شده، لینک منابع رسمی، تست‌های اجراشده، موارد Unknown و ریسک‌های باقی‌مانده را داشته باشد.
22. CI را تا پایان مانیتور کن. اگر شکست خورد Log را بخوان، علت ریشه‌ای را اصلاح، Commit و Push کن. تا سبز شدن CI متوقف نشو.

## قواعد غیرقابل‌مذاکره

- ساختن نتیجه، حدس زدن، تعمیم یک ISP به کل ایران و اعلام موفقیت بدون درخواست واقعی ممنوع است.
- استفاده از VPN مجاز است، اما جا زدن نتیجهٔ VPN به‌عنوان دسترسی مستقیم ایران ممنوع است.
- دور زدن تحریم، KYC، شماره تلفن، پرداخت، CAPTCHA یا Terms ممنوع است.
- چاپ یا Commit هر نوع Secret و اطلاعات شخصی ممنوع است.
- حذف تست، `--no-verify`، Force Push به `main` و Commit مستقیم روی `main` ممنوع است.
- تغییر خارج از Scope، Refactor نمایشی و افزودن Dependency بدون ضرورت ممنوع است.
- اگر چیزی قابل اثبات نیست، صریحاً `unknown` بگذار؛ Unknown دقیق بهتر از پاسخ قطعی ساختگی است.

## معیار پایان

کار فقط وقتی تمام است که:

- هر یازده Provider دارای نتیجهٔ مستند یا دلیل روشن برای Unknown باشند؛
- حداقل دو شبکهٔ واقعی ایران بررسی شده باشند، یا مانع دقیق و اثبات‌پذیر ثبت شده باشد؛
- هیچ Secret/IP کامل در Git و خروجی عمومی نباشد؛
- داده‌ها، منابع و README همگام باشند؛
- `npm test` و CI سبز باشند؛
- Draft PR با گزارش شفاف ساخته شده باشد؛
- پیام نهایی شامل URL شاخه، Commit، PR، نتیجهٔ CI، جدول وضعیت Providerها و فهرست دقیق موانع باقی‌مانده باشد.

اگر کلید یا تعامل انسانی در دسترس نبود، تمام کارهای مستقل را کامل کن، یک Checklist دقیق از فقط همان ورودی‌های مسدودکننده بساز و منتظر همان ورودی بمان؛ تحلیل کلی یا پیشنهاد مبهم تحویل نده.
