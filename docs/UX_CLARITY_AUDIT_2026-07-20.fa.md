# ممیزی وضوح تجربه کاربری و رشد — ۲۰۲۶-۰۷-۲۰

## سیگنال مسئله

در تست غیررسمی با دو گروه، کاربران عادی اعلام کردند که کاربرد سایت را متوجه نشده‌اند و کاربران برنامه‌نویس نیز مسیر استفاده را گیج‌کننده دانستند. این بازخورد نشان می‌دهد مسئله اصلی فعلی کمبود داده یا قابلیت نیست؛ مسئله، وضوح ارزش محصول، معماری اطلاعات و شروع کار است.

## تشخیص اصلی

صفحه نخست هم‌زمان نقش‌های زیر را اجرا می‌کند:

- معرفی محصول؛
- کاتالوگ ۲۲ Provider؛
- موتور فیلتر حرفه‌ای؛
- پیشنهادگر خودکار؛
- راهنمای SEO؛
- فهرست صفحات Provider؛
- FAQ؛
- دعوت به مشارکت GitHub.

کاربر قبل از فهمیدن پاسخ سه سؤال پایه با واژگان تخصصی روبه‌رو می‌شود:

1. این سایت دقیقاً چه کاری انجام می‌دهد؟
2. برای چه کسی ساخته شده است؟
3. قدم بعدی من چیست؟

## مشکلات اولویت صفر

### ۱. پیام ارزش بیش از حد فنی است

عبارت‌هایی مانند LLM، ماشین‌خوان، OpenAI-compatible، Provider، Direct، VPN، Signup، RPM و RPD برای کاربر تازه‌وارد بدون توضیح نمایش داده می‌شوند.

### ۲. دو مسیر اصلی از هم جدا نشده‌اند

کاربر عادی به توضیح ساده و مثال نیاز دارد؛ توسعه‌دهنده به انتخاب سریع، کد شروع و اطلاعات فنی. صفحه فعلی هر دو را هم‌زمان ارائه می‌کند و برای هیچ‌کدام مسیر روشن نمی‌سازد.

### ۳. دعوت به اقدام اصلی مبهم است

«مشاهده APIها» کاربر را مستقیماً به فهرست و فیلترهای پیچیده می‌برد. ابزار هدایت‌شده API Finder باید مسیر اصلی باشد و کاتالوگ به‌عنوان مسیر حرفه‌ای ثانویه معرفی شود.

### ۴. بار شناختی بالا است

آمار، پنج فیلتر، پیشنهادگر، مرتب‌سازی، کارت‌های مفصل و اصطلاحات متعدد در یک جریان متوالی نمایش داده می‌شوند. امکانات حرفه‌ای باید با Progressive Disclosure ارائه شوند.

### ۵. سلسله‌مراتب محتوا با هدف رشد هم‌راستا نیست

بخش آموزش انگلیسی در صفحه فارسی، فهرست طولانی Providerها و CTA مشارکت قبل از تثبیت درک محصول، تمرکز کاربر را از مسیر اصلی منحرف می‌کنند.

## معماری اطلاعات پیشنهادی

### سطح اول — فهم محصول

- عنوان ساده: «API رایگان هوش مصنوعی برای پروژه‌ات پیدا کن»
- توضیح یک‌خطی: «این سایت سرویس‌هایی را مقایسه می‌کند که برنامه یا ربات شما را به مدل‌های هوش مصنوعی وصل می‌کنند.»
- مثال ملموس: چت‌بات، تولید متن، کدنویسی، جست‌وجوی معنایی.

### سطح دوم — انتخاب مسیر

- مسیر تازه‌کار: «API چیست و از کجا شروع کنم؟»
- مسیر برنامه‌نویس: «برای پروژه‌ام API انتخاب کن»
- مسیر حرفه‌ای: «همه سرویس‌ها و فیلترهای پیشرفته»

### سطح سوم — انجام کار

- API Finder به‌عنوان Primary CTA؛
- سه مرحله: نیازت را انتخاب کن، پیشنهادها را مقایسه کن، مستندات و نمونه کد را باز کن؛
- کاتالوگ و فیلترها به‌صورت ثانویه و قابل بازشدن.

### سطح چهارم — اعتماد و جزئیات

- وضعیت ایران با توضیح ساده؛
- تاریخ بررسی؛
- منبع رسمی؛
- واژه‌نامه کوتاه؛
- روش امتیازدهی و محدودیت‌ها در disclosure.

## برنامه اجرا

### فاز ۱ — Clarity Refactor

- بازنویسی Hero و Metadata به زبان ساده؛
- ساخت دو مسیر «تازه‌کار» و «برنامه‌نویس»؛
- افزودن بخش «چطور کار می‌کند؟»؛
- تبدیل API Finder به CTA اصلی؛
- قرار دادن فیلترهای پیشرفته در disclosure؛
- حذف بخش آموزش انگلیسی از جریان اصلی فارسی و جایگزینی با لینک زبان؛
- افزودن واژه‌نامه کوتاه؛
- حفظ تمام IDهای لازم برای JavaScript و تست‌ها.

### فاز ۲ — Developer Activation

- صفحه شروع سریع مستقل؛
- نمونه درخواست Copy-ready برای Python، Node.js و curl؛
- انتخاب Provider بر اساس کاربرد، نه اصطلاحات داخلی Catalog؛
- نمایش واضح الزامات ثبت‌نام، کارت بانکی و دسترسی ایران؛
- CTA «اولین درخواست را بساز».

### فاز ۳ — Usability and Growth Measurement

- رویدادهای Analytics برای انتخاب مسیر، شروع Finder و کلیک مستندات؛
- تست وظیفه‌محور با کاربران عادی و برنامه‌نویس؛
- سنجش نرخ فهم پیام در ۵ ثانیه؛
- سنجش Task Success برای یافتن API مناسب؛
- ثبت Search Console CTR و Query Intent قبل و بعد از تغییر.

## معیارهای پذیرش

- کاربر بدون دانش فنی بتواند در پنج ثانیه هدف سایت را توضیح دهد؛
- اولین CTA کاربر را به مسیر هدایت‌شده ببرد؛
- اصطلاحات فنی قبل از استفاده توضیح داده شوند؛
- فیلترهای حرفه‌ای به‌صورت پیش‌فرض بر صفحه غالب نباشند؛
- مسیر تازه‌کار و توسعه‌دهنده از بالای صفحه قابل تشخیص باشند؛
- SEO metadata همچنان intentهای «API رایگان هوش مصنوعی»، «API رایگان LLM» و «API برای ایران» را پوشش دهد؛
- `npm test`، `site:build`، `site:check`، Strict SEO و Regression همگی پاس شوند.
