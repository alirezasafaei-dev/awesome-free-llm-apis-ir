# راهنمای انتخاب API رایگان مدل زبانی؛ از Free Tier واقعی تا محدودیت‌های دسترسی

## مقدمه

تصور کنید یک API رایگان برای مدل زبانی پیدا کرده‌اید، ثبت‌نام می‌کنید، کد می‌نویسید و بعد از یک هفته پیغام «اعتبار شما تمام شده» ظاهر می‌شود. این اتفاق رایجی است چون APIهایی که خود را «رایگان» معرفی می‌کنند، تعریف یکسانی از این کلمه ندارند.

این مقاله به شما کمک می‌کند قبل از انتخاب، معیارهای درست را بشناسید و بر اساس نیاز واقعی تصمیم بگیرید.

## تفاوت Free Tier، Trial و Credit

APIهای رایگان معمولاً در یکی از این دسته‌ها قرار می‌گیرند:

**Free Tier دائمی:** سهمیه مشخصی است که هر ماه یا هر روز بدون پرداخت تمدید می‌شود. مثلاً یک سرویس ممکن است روزانه ۱۰۰۰ درخواست رایگان بدهد تا همیشه. سرویس‌هایی مانند Google Gemini API یا GitHub Models تا سقف معینی در این دسته قرار می‌گیرند.

**Trial موقت:** دسترسی محدود به زمان یا حجم مشخص که پس از اتمام نیاز به ارتقا دارد. مثلاً ۳۰ روز دسترسی رایگان یا $۱ اعتبار اولیه.

**Credit اولیه:** مقدار مشخصی توکن یا اعتبار که پس از مصرف تمام می‌شود. ممکن است سرویس پس از آن هزینه دریافت کند یا دسترسی را محدود کند.

نکته مهم: بعضی سرویس‌ها در مستندات خود را «رایگان» معرفی می‌کنند اما در عمل فقط Trial یا Credit محدود هستند. مطالعه دقیق بخش Pricing مستندات رسمی ضروری است.

## محدودیت‌های RPM، RPD و TPM

حتی در Free Tierهای دائمی، محدودیت نرخ درخواست وجود دارد:

- **RPM (Request Per Minute):** چه تعداد درخواست در هر دقیقه مجاز است؟ برای کارهای real-time مهم است.
- **RPD (Request Per Day):** سقف روزانه چقدر است؟ برای پردازش دسته‌ای باید محاسبه شود.
- **TPM (Token Per Minute):** سرعت مصرف توکن چقدر می‌تواند باشد؟

برای مثال، یک API ممکن است ۱۰ RPM / ۱۰۰۰ RPD داشته باشد یعنی روزی حدود ۱۶ ساعت کار مداوم با نرخ کامل ممکن نیست. این محدودیت‌ها را باید با workload خود تطبیق دهید.

## نیاز به کارت بانکی

یکی از بزرگ‌ترین موانع برای توسعه‌دهندگان در ایران، الزام به وارد کردن کارت بانکی برای ثبت‌نام است. بسیاری از سرویس‌های معروف حتی برای Free Tier هم کارت می‌خواهند.

خوشبختانه چندین Provider معتبر بدون کارت بانکی هم ثبت‌نام می‌پذیرند. معمولاً محدودیت نرخ در این سرویس‌ها پایین‌تر است اما برای شروع کار، نمونه‌سازی اولیه و آموزش گزینه مناسبی هستند.

## سازگاری با OpenAI SDK

اغلب Providerهای جدید از پروتکل سازگار با OpenAI استفاده می‌کنند. یعنی می‌توانید با تغییر Base URL در کتابخانه `openai` پایتون یا جاوااسکریپت، بین سرویس‌های مختلف جابه‌جا شوید:

```python
from openai import OpenAI
client = OpenAI(base_url="BASE_URL_HERE", api_key="YOUR_KEY")
response = client.chat.completions.create(model="model-name", messages=[...])
```

اگر Providerای Base URL شفاف و مستندات OpenAI-compatible داشته باشد، جابه‌جایی بین سرویس‌ها و مقایسه کیفیت خروجی ساده‌تر می‌شود.

## وضعیت دسترسی از ایران

این بخش شاید مهم‌ترین معیار برای توسعه‌دهندگان داخل کشور باشد. برای بررسی دسترسی یک API از ایران باید سه منبع اطلاعات را جداگانه بررسی کرد:

۱. **مستندات رسمی Provider:** آیا در شرایط استفاده یا Geo-restrictions اشاره‌ای به ایران شده؟
۲. **تست مستقیم:** آیا از یک IP ایران، درخواست موفق به API ارسال می‌شود؟
۳. **تست با VPN:** آیا با تغییر موقعیت منطقه‌ای قابل استفاده است؟

این سه دسته را نباید با هم ترکیب کرد. نتیجه تست VPN به معنی دسترسی مستقیم نیست. همچنین «نامشخص» بودن وضعیت به معنی کار کردن سرویس نیست؛ فقط یعنی شواهد کافی برای نتیجه‌گیری وجود ندارد.

## چطور اطلاعات APIها را به‌روز نگه داریم

سیاست‌های Free Tier و محدودیت‌های منطقه‌ای مرتب تغییر می‌کنند. یک Provider ممکن است امروز رایگان باشد و فرداً سهمیه را کاهش دهد. بهترین روش:

- همیشه مستندات رسمی را چک کنید نه پست‌های قدیمی
- قبل از وابسته شدن به یک API، محدودیت‌ها را در workload واقعی تست کنید
- از منابعی استفاده کنید که تاریخ آخرین بررسی را ثبت می‌کنند

## منابع مفید

- مستندات رسمی هر Provider بهترین منبع برای اطلاعات Pricing است
- Catalog JSON برخی پروژه‌های متن‌باز ممکن است اطلاعات مقایسه‌ای مفیدی داشته باشند
- انجمن‌های تخصصی و issues گیت‌هاب منابع خوبی برای تجربه واقعی کاربران هستند
