روش‌شناسی، حریم خصوصی و اصلاح داده

این کاتالوگ میان سیاست رسمی، مشاهدهٔ شبکهٔ مستقیم ایران، مشاهدهٔ VPN و اعتبارسنجی حساب تفاوت می‌گذارد. نبود شواهد هرگز با حدس پر نمی‌شود.

منبع و تاریخ هر ادعا

برای سهمیه، قیمت، مدل، احراز هویت و سیاست منطقه‌ای، مستندات first-party Provider مبناست. هر رکورد Provider تاریخ آخرین بررسی و منبع خود را دارد؛ برای تصمیم نهایی همیشه صفحهٔ اختصاصی Provider و لینک‌های رسمی آن را بررسی کنید.

پایش freshness فقط نشان می‌دهد که یک رکورد نیازمند بازبینی نشده است؛ جایگزین بررسی منبع رسمی در زمان تصمیم production نیست. در صورت تعارض میان مستندات Provider و یک مشاهدهٔ فنی، هر دو evidence جدا نگه داشته می‌شوند.

مشاهده کاتالوگ و صفحات Provider

مرز شواهد دسترسی ایران

موفقیت از یک host خارجی یا VPN، مدرک دسترسی مستقیم ایران نیست. وضعیت مستقیم فقط با نتیجهٔ sanitize‌شده، تاریخ‌دار و حاصل از مسیر مستقیم ایران ثبت می‌شود. این پروژه راه دورزدن تحریم، KYC، CAPTCHA، پرداخت یا Terms را ارائه نمی‌کند.

روش‌شناسی کامل در مخزن · قرارداد راستی‌آزمایی ایران

حریم خصوصی و Analytics

دادهٔ عمومی نباید شامل API key، cookie، ایمیل، شماره تلفن، IP کامل، hostname، شناسهٔ حساب، header احراز هویت یا بدنهٔ خام پاسخ باشد. Analytics فقط برای eventهای محصولیِ حداقلی و بدون دادهٔ هویتی طراحی شده است؛ آینهٔ ایران analytics مستقل ندارد.

اصلاح و گزارش خطا

اگر دادهٔ کاتالوگ نادرست، قدیمی یا ناقص است، Issue عمومی ثبت کنید. گزارش باید فقط شامل Provider، تاریخ، نوع شواهد و نتیجهٔ sanitize‌شده باشد. اصلاح‌ها با منبع و تاریخ بررسی می‌شوند؛ گزارش ناقص یا حساس منتشر نمی‌شود.

شرایط استفاده

کاتالوگ یک منبع اطلاع‌رسانی است و تضمین دسترسی، سهمیه یا تداوم سرویس هیچ Providerی نیست. پیش از استفادهٔ production، Terms، قیمت، نگهداری داده و سیاست منطقه‌ای همان Provider را در منبع رسمی بررسی کنید.
