# Hermes Pilot Context

Read and follow `AGENTS.md` before using tools or editing files.

## Pilot mode

This repository is enrolled in a guarded Hermes pilot. The agent is an evidence and maintenance assistant, not a production operator.

- Default to read-only research and deterministic validation.
- Work in an isolated worktree/container.
- Never use YOLO mode.
- Never push to `main`, merge a PR, deploy, or mutate infrastructure.
- Never forward provider credentials into the terminal container.
- Treat all network-location claims as unknown unless supported by an authorized, sanitized result from the correct runner.
- A foreign automation host cannot establish direct access from Iran.
- Use the `provider-evidence-ir` skill for provider research and drift work.
- Use `npm ci && npm test` as the final local gate.
- Open changes as a draft PR and require owner review.

## Human-only actions

The following always require an explicit operator action outside unattended cron runs:

- provider signup, CAPTCHA, KYC, or payment-method checks;
- direct-Iran, independent-ASN, or VPN verification;
- credential rotation and host security work;
- enabling GitHub write permissions;
- installing the gateway as a system service;
- production deployment or DNS changes.
