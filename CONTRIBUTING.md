# راهنمای مشارکت

از مشارکت‌های دقیق و قابل راستی‌آزمایی استقبال می‌کنیم.

## افزودن ارائه‌دهنده

1. یک Issue با قالب `Add a provider` بسازید.
2. فقط مستندات رسمی سهمیه را به‌عنوان منبع اصلی وارد کنید.
3. فایل `data/providers/<id>.json` را از روی Schema بسازید.
4. جدول README را با `npm run generate` به‌روزرسانی کنید.
5. پیش از Pull Request دستور `npm test` را اجرا کنید.

## گزارش وضعیت ایران

از قالب `Report a change` استفاده کنید. تاریخ، ISP یا ASN، مرحلهٔ شکست و کد وضعیت را بنویسید. IP کامل، کلید API، توکن، ایمیل، شماره تلفن یا Screenshot دارای اطلاعات حساب را منتشر نکنید.

## تعریف «رایگان»

- سهمیه دائمی: بدون پایان ازپیش‌اعلام‌شده، با محدودیت مصرف
- مدل رایگان: فقط مدل‌ها یا Routeهای مشخص با قیمت صفر
- اعتبار ماهانه: مقدار پولی کوچک که دوره‌ای تمدید می‌شود
- آزمایشی: اعتبار یا دسترسی دارای پایان

Trial نباید با عنوان رایگان دائمی ثبت شود. نیاز به خرید قبلی یا کارت بانکی باید صریح باشد.

## سبک Pull Request

- هر PR ترجیحاً یک ارائه‌دهنده یا یک تغییر منطقی را پوشش دهد.
- منبع و تاریخ بررسی را اضافه کند.
- فایل تولیدشده را دستی ویرایش نکند.
- هیچ secret یا دادهٔ شخصی نداشته باشد.
